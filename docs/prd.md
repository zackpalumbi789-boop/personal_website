# 需求文档

## 1. 应用概述

- **应用名称**：陆祁的个人主页
- **应用描述**：一个面向朋友与面试官的个人主页，包含个人信息展示区与数字分身聊天区，整体风格现代清爽、具备科技感，支持移动端适配。

---

## 2. 用户与使用场景

### 2.1 目标用户
- 朋友：了解陆祁近况与个人状态
- 面试官：快速了解陆祁的背景、能力与个人特点

### 2.2 核心使用场景
- 访客打开主页，浏览个人信息，形成初步印象
- 访客通过数字分身聊天区提问，获取关于陆祁的个性化回答

---

## 3. 页面结构与核心功能

### 3.1 页面总览

```
个人主页（单页滚动）
├── 顶部 Hero 区
│   ├── 头像
│   ├── 姓名
│   └── 一句话介绍
├── 个人信息展示区
│   ├── 当前状态
│   └── 兴趣方向
└── 数字分身聊天区
    ├── 聊天展示窗口
    └── 用户输入框
```

### 3.2 顶部 Hero 区

**功能说明**：
- 展示头像（圆形头像占位，支持后续替换图片）
- 展示姓名：陆祁
- 展示一句话介绍：「我是一名互联网从业近17年的老兵，现在打算拥抱AI」
- 整体采用蓝色系渐变背景，配合轻量粒子或光线动效，营造科技感氛围

### 3.3 个人信息展示区

**功能说明**：
- 以卡片或信息块形式展示以下内容：
  - **当前在做**：忙着找工作，忙着学AI
  - **擅长方向**：AI、解决业务问题
  - **兴趣爱好**：学习新知识（AI、理财）、尝试新体验
  - **个人标签**：踏实
- 信息块之间保持清晰间距，视觉层次分明
- 支持移动端单列布局，桌面端双列或多列布局

### 3.4 数字分身聊天区

**功能说明**：
- 区域标题：「和我的数字分身聊聊」或类似科技感文案
- 聊天窗口展示对话气泡，区分「访客提问」与「数字分身回答」两种气泡样式
- 数字分身基于以下预设知识库进行回答：
  - 职业身份：互联网技术总监，刚刚被裁员
  - 近期状态：忙着找工作，忙着学AI
  - 擅长方向：AI、解决业务问题
  - 个人特点：踏实
  - 兴趣：学习新知识（AI、理财）、尝试新体验
- 预置3条快捷提问按钮，点击后自动填入输入框：
  - 「你擅长什么？」
  - 「你现在在做什么？」
  - 「你有什么特点？」
- 用户在输入框输入问题后，点击发送按钮或按回车键提交
- 数字分身回答以打字机逐字输出动效呈现，增强科技感
- 聊天记录仅在当前会话内保留，刷新页面后重置
- 输入框为空时禁止发送

---

## 4. 业务规则与逻辑

| 规则编号 | 规则描述 |
|---|---|
| BR-01 | 数字分身回答内容基于预设知识库生成，不调用外部 AI 接口，采用关键词匹配或预设问答对实现 |
| BR-02 | 快捷提问按钮点击后，问题自动填入输入框并立即发送 |
| BR-03 | 打字机动效期间，发送按钮置灰，防止重复提交 |
| BR-04 | 页面为单页滚动结构，各区块依次排列，无跳转页面 |
| BR-05 | 移动端适配：所有区块在小屏幕下自动调整为单列布局，聊天区输入框固定在聊天区底部 |

---

## 5. 异常与边界情况

| 场景 | 处理方式 |
|---|---|
| 用户输入空内容点击发送 | 发送按钮不可点击，输入框轻微抖动提示 |
| 用户输入内容超过200字 | 输入框限制最大输入长度为200字，超出部分不可继续输入 |
| 数字分身无法匹配问题 | 返回默认回答：「这个问题有点难住我了，你可以换个方式问问我～」 |
| 聊天记录过长 | 聊天窗口内部滚动，不影响页面整体布局 |

---

## 6. 验收标准

- 页面可在主流浏览器（Chrome、Safari）正常访问与展示
- 移动端（375px 宽度基准）布局无错位，内容完整可读
- 顶部 Hero 区头像、姓名、一句话介绍正确展示
- 个人信息展示区四项内容（当前在做、擅长方向、兴趣爱好、个人标签）完整展示
- 数字分身聊天区可正常输入、发送、接收回答
- 快捷提问按钮点击后可自动发送对应问题
- 打字机动效正常触发，动效期间发送按钮置灰
- 整体配色以蓝色系为主，视觉风格现代清爽，具备科技感
